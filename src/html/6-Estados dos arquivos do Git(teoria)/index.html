Teoria sobre o estado dos arquivos do GitHub

Os arquivos do Git tem alguns estados

  • Untracked: quando o arquivo ainda não é conhecido pelo git;
  • Tracked: é quando o arquivo já é conhecido pelo git, e ele pode ser: Unmodified, Modified, Staged:
    • Unmodified: arquivo já é conhecido pelo git, mas não foi modificado;
    • Modified: arquivo já é conhecido pelo git, mas modificado;
    • Staged: marcou um arquivo modificado para ir para o próximo snapshot de commit.

Quando colocamos o arquivo no Git ele começa como Untracked.Quando fazemos o git add ele fica Staged.Fazemos então o git commit ele fica Unmodified e está meio que pronto, mas tem três opções do que fazer agorá.

1:Só simplesmente deixar ele assim até que alguém perceba algum erro.Se alguém achar algum erro fazemos um edit the file(modificação) agorá ele está no estado Modified.Desse modo fazemos um git add e ele volta a ficar Staged;

2:Fazer atualizações e ele ficará Modified;

3:Podemos também Remove the file(remover o arquivo) e ele ficará Untracked.
