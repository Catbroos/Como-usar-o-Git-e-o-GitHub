Prática sobre o estado dos arquivos do GitHub

Parte1:Configuração desse repositório

Primeiro abra a pasta onde está o arquivo no Git.Importante que se certifique que está na pasta certa!

Passo1:Criamos um repositório Git lá com o comando "git init" e agorá podemos fazer o ciclo de vida dos arquivos.

Passo2:Mudamos o nome do master para main usando o comando "git branch -M main"

Passo3:Lincar o Git com o repositório do GitHub usando o comando "git remote add origin (link do repositório do GitHub em https).git".Caso queira ver se funcionou use o comando "git remote -v"

Parte2:Fazendo o ciclo de vida dos aquivos.

Dica use o comando "git status" para ver o status dos arquivos se ele está Untracked, Unmodified, Modified, Staged.

Passo4:Tranforme todos os arquivos Untracked em Staged usando o comando "git add (nome do arquivo ou .(que transforma todos os arquivos))".

Passo5:Comite as coisas dando um "git commit -m "(Uma mensagem doque foi comitado)"".

Condições:

Passo6:Muito provavel mente você terá que fazer alterações nos arquivos.Então sempre que fizer isso de um "git add .", "git status", "git commit -m (mensagem)" e um "git status".

Passo7:Caso queira deletar um arquivo mandeo para a lixeira.

Finalização:

Passo8:Mandando para o GitHub

Agorá estamos com tudo pronto para mandar para o GitHub mandamos o comando "git push -u origin main".
